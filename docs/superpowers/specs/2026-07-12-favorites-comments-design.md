# お気に入り・コメント機能 設計書

日付: 2026-07-12
ステータス: ユーザー承認済み（設計段階）

## 目的

スカウティングレポートWebアプリで、チームメンバーがアーティストに☆お気に入りを
付けたり、コメント（キャスティング会議のメモ想定）を残せるようにする。誰が
付けたかも分かり、即座に他メンバーへ反映される。

## 決定事項（ブレスト結果）

| 論点 | 決定 |
|---|---|
| 共有範囲 | チーム全員で共有（誰が付けたかも分かる） |
| ユーザー識別 | 初回にニックネームを入力しブラウザ(localStorage)に保存。ログイン機構なし、自己申告制 |
| 書き込み方式 | Google Apps Script（GAS）をWebアプリとして公開し、専用Googleシートに追記 |
| 反映タイミング | **即時**（名簿本体は週次のまま、お気に入り・コメントだけライブ取得に分離） |
| コメント件数 | 1人が複数件（時系列で積み上げ）。削除・編集はv1では作らない |
| お気に入りの動作 | トグル式（同じ人が同じアーティストに再度押すと解除） |
| 表示場所 | 詳細ビュー（☆ボタン＋コメント欄）＋名簿一覧カードにも★マーク |
| 不正書き込み対策 | GAS側でアプリの合言葉を検証してから読み書きを許可 |
| データの機密性 | 平文で問題ない（名簿本体のようなAES暗号化はしない。合言葉ゲート＋HTTPSで十分と判断） |

## 全体アーキテクチャ

名簿データ（roster/candidates/history/stats）の流れは**変更しない**（週次ビルド→
data.enc→GitHub Pages、[2026-07-10設計](2026-07-10-scouting-webapp-design.md)のまま）。

お気に入り・コメントは新しい独立系統として、ブラウザが直接GASと通信する：

```
専用Googleシート「Favorites」（新規タブ、既存Scouting reportシート内）
  列: timestamp | name(ニックネーム) | artist(アーティスト名) | type(fav/comment) | text

GAS Webアプリ（doGet/doPost、単一ファイル gas/Code.gs）
  doGet(passphrase)  → 合言葉検証 → 全件をJSON配列で返す
  doPost(passphrase, name, artist, type, text)
                     → 合言葉検証 → 1行追記（fav は既存トグルなら削除、なければ追加）

ブラウザ（site/app.js）
  - ロック解除時に保持している合言葉をそのままGAS呼び出しにも使う（別入力不要）
  - 詳細ビューを開く／「更新」ボタン押下のたびに doGet で最新を取得
  - ☆押下・コメント投稿は doPost → 成功したら再度 doGet して表示を更新
  - data.enc には一切含めない。取得失敗時は「お気に入り・コメントを読み込めませんでした」
    と表示し、名簿本体の閲覧は妨げない（独立して失敗できる設計）
```

## データ構造

Googleシート「Favorites」タブの1行1件、時系列で積み上がる：
```
timestamp | name | artist | type(fav|comment) | text
```
- `type=fav` の行はトグル用: 同じ `name`+`artist` の `fav` 行が既にあれば削除、
  なければ追加する（GAS側で判定）
- `type=comment` の行は追記のみ（削除・編集なし）。`text` にコメント本文

GASの `doGet` レスポンス（JSON）：
```json
{
  "favorites": [{"name": "アラタ", "artist": "Zeroko", "timestamp": "..."}],
  "comments": [{"name": "アラタ", "artist": "Zeroko", "timestamp": "...", "text": "..."}]
}
```

## UI

- **初回訪問**（ロック解除直後、初回のみ）: 「お名前を入力してください」のポップアップ。
  入力後 localStorage に保存し、以後は自動で使う
- **ヘッダー**: 現在のニックネームを小さく表示し、クリックすると再入力できる
  （名前を変更したい場合はここから変更）
- **詳細ビュー**:
  - ☆ボタン: 自分が押していれば塗り、押していなければ枠のみ。「☆ 3人」のように
    人数も表示
  - コメント欄: 入力フォーム＋下に時系列一覧（ニックネーム・日時・本文）
  - 「更新」ボタン: doGet を再実行して最新を反映
- **名簿一覧カード**: 誰か1人でもお気に入りしていれば★マークをカードに追加表示
  （合計人数は出さない、シンプルな目印のみ）

## エラー処理

- doGet/doPost の通信失敗・タイムアウト: 該当箇所に「読み込めませんでした」を
  表示し、他の機能（名簿閲覧・検索）はブロックしない
- 合言葉不一致（GAS側）: 401相当のレスポンスを返し、フロントは同じエラー表示に
  フォールバック（通常は起こらない想定。ロック画面を通過している時点で正しい
  合言葉を持っているため）

## 実装範囲

- **新規**: `gas/Code.gs`（GAS単一ファイル、リポジトリには手順書として置くのみ。
  実際のデプロイはGoogle側で行うため、コード自体はリポジトリで管理してもリスクなし）
- **フロント**: `site/app.js` にニックネーム管理・☆ボタン・コメントUI・
  doGet/doPost 呼び出しを追加。GAS の公開URLは `site/app.js` 内の定数として持つ
  （合言葉検証があるので、URL自体が漏れても書き込みはできない）
- **既存の週次ビルド・暗号化・GitHub Pages配信の枠組みは変更なし**
- Node側（tools/, tests/）の変更はなし（お気に入り・コメントはビルドパイプラインを
  通らないため）

## テスト・検証

- GAS単体: doGet（空配列が返る）、doPost（fav追加→doGetで確認→同条件で再度
  doPost→トグル解除確認）、コメント追記、合言葉不一致時の拒否
- ブラウザ: 初回ニックネーム入力→保存確認、☆トグルの見た目切り替え、
  コメント投稿→一覧反映、「更新」ボタンでの再取得、名簿カードの★マーク表示、
  GAS通信失敗時に名簿閲覧が壊れないこと、スマホ幅（375px）で崩れないこと
